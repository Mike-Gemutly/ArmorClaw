## 4. Server Name Conflict Handling

### quickstart-entrypoint.sh (lines 210-242)
```bash
EXISTING_SERVER_NAME=$(docker exec "$CONDUIT_CONTAINER" cat /etc/conduit.toml 2>/dev/null | ...)

if [ -n "$EXISTING_SERVER_NAME" ]; then
    log "Found existing server name in database: $EXISTING_SERVER_NAME"
    
    if [ "$SERVER_NAME_USER_PROVIDED" = true ] && [ "$ARMORCLAW_SERVER_NAME" != "$EXISTING_SERVER_NAME" ]; then
        log_warn "Provided server name ($ARMORCLAW_SERVER_NAME) differs from existing database ($EXISTING_SERVER_NAME)"
        log_warn "Conduit cannot reuse database after changing server name"
        
        # Delete the old database to start fresh with new server name
        log "Deleting existing database to apply new server name..."
        docker rm -f "$CONDUIT_CONTAINER" >/dev/null 2>&1 || true
        docker volume rm armorclaw-conduit-data >/dev/null 2>&1 || true
        ...
    else
        # Use existing server name from database
        ARMORCLAW_SERVER_NAME="$EXISTING_SERVER_NAME"
        ...
    fi
fi
```

**Analysis:**
- Reads server_name from existing Conduit config
- Warns if user provides different server name
- Option 1: Delete and recreate with new name (user explicitly requested)
- Option 2: Use existing name to preserve database

**Evidence Quality: EXCELLENT**
- Prevents database corruption from server name mismatch
- User is warned before destructive operation
- Smart default: preserve existing data unless explicitly overridden

## 7. Identified Issues

### Issue 1: Lockfile Location in /tmp

**Severity:** LOW
**Location:** installer-v5.sh:106, setup-quick.sh:53, setup-matrix.sh:39

**Analysis:**
- Lockfiles created in `/tmp/` which is cleared on reboot
- If installer crashes and system reboots, stale lock won't exist
- However: `/tmp/` is standard location for temporary locks
- flock automatically releases lock on process exit

**Recommendation:** Acceptable - flock semantics handle process termination

### Issue 2: No Cleanup of Partial State on Failure
**Severity:** MEDIUM
**Location:** All setup scripts

**Analysis:**
- If installation fails mid-way, partial state may remain
- Example: Bridge installed but service not created
- However: Scripts are designed to be re-run safely

**Recommendation:** Consider adding rollback-on-failure for critical steps
